1. O que é um repositório no Git e qual a sua importância?
R: Repositórios git é onde o código-fonte e seu histórico de versões são armazenados. Ele é importante para ter controle da versão, colaboração em equipe, segurança e automatizacão do desenvolvimento. 

2. Qual a função do Git Bash?
R: O Git Bash é um terminal que permite executar comandos git no Windows simulando um ambiente Unix/Linux para facilitar o uso do Git.

3. O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?
R: Git add adiciona arquivos para o commit. O Git add <arquivo> adiciona um arquivo específico. Já o Git add . adiciona todos os arquivos modificados no diretório. 

4. Por que utilizamos o ponto (.) no comando git add .?
R: Usamos par adicionar todas as mudanças no diretório. 

5. Explique o comando git commit e por que é importante criar uma mensagem descritiva.
R: O Git commit serve para dar uma discrição da alteração feita no projeto. É importante para sabermos o que foi feito exatamente naquele ponto do projeto

6. Qual a função do comando git push?
R: O Git Push serve para enviar os arquivos ao repositório do GitHub.

7. Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?
R: É importante para não nos perdemos no processo de criação dos projetos e para sempre registrarmos as atualizções.

8. Explique a função das tags HTML e como elas estruturam uma página.
R: As tags estruturam e organizam o conteúdo de uma página web. Elas indicam ao navegador como exibir textos, imagens, links e outros elementos.

9. O que são atributos em uma tag HTML e qual a sua função?
R: Os atributos em uma tag HTML são informações extras que configuram ou modificamno comportamento da tag. Eles sempre aparecem dentro da tag de abertura.

10. O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?
R: É uma ferramenta integrada na maioria dos editores de código, como VS code, Subblime Text e Atom, que ajuda a escrever HTML e CSS de forma muito mais rápida. Ele usa abreviações que se expandem automaticamente em blocos completos de código.

11. O que é o box-sizing no CSS e por que usamos ele no reset inicial?
R: Box-sizing é uma propriedade do CSS que define como a largura e altura de um elemento devem ser calculadas.

12. Por que é importante utilizar tags semânticas no HTML?
R: É importante porque elas dão significado ao conteúdo da página, não só para os navegadores, mas também para buscadores e tecnologias assistivas(como leitores de tela).

13. O que são media queries no CSS e qual a sua função?
R: São regras do CSS que permitem adaptar o estilo de uma página conforme o tamanho da tela ou características do dispositivo, como largura, altura, orientação e até resolução.

14. Qual a função do GitHub Pages?
R: Sua função é hospedar sites estáticos(HTML, CSS, JavaScript) diretamente de um repositório.

15. O que é o Flexbox no CSS e para que ele é utilizado?
R: É uma ferramenta usada para criar layouts flexíveis e responsivos com mais facilidade. Ele organiza os elementos dentro de um contêiner de forma eficiente, mesmo quando o tamanho da tela ou o conteúdo muda.

16. Qual a diferença entre justify-content e align-items em um container Flexbox?
R: A diferença entre eles está na direção em que eles atuam. Justify-content - alinhamento horizontal, Align-items - alinhamento vertical.

17. Como o flex-grow funciona e qual o seu propósito no layout Flexbox?
R: Ele Define a habilidade de um flex item de crescer, caso necessário 

18. Explique a diferença entre flex-direction: column; e flex-direction: row;.
R: A diferença entre eles é que o column organiza os elementos em coluna e o row organiza em linha.
